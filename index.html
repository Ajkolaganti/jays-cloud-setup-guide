01
AWS EC2 Instance
1
Launch a t3.micro Ubuntu instance

Go to EC2 → Launch Instance in the AWS Console. Use these settings:

FieldValue
AMIUbuntu Server 24.04 LTS (HVM), SSD Volume
Instance typet3.micro
Key pairCreate new (.pem file — store it safely)
Storage30 GB gp3
Security GroupInbound: SSH 22, HTTP 80, HTTPS 443
2
Allocate & associate an Elastic IP

EC2 → Elastic IPs → Allocate → Associate with your instance. This gives you a stable public IP that survives reboots.

⚠Copy your Elastic IP — you'll point DNS A records at it for every subdomain.
3
Add a 4 GB swap file

t3.micro ships with 1 GB RAM. Swap prevents OOM kills during package installs.

bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
02
Connect via SSH
4
SSH from Windows / PowerShell
powershell
# Fix key permissions on Windows
icacls C:\path\to\your-key.pem /inheritance:r /grant:r "%USERNAME%:R"

# Connect
ssh -i C:\path\to\your-key.pem ubuntu@<YOUR_ELASTIC_IP>
5
Update system & install essentials
bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git unzip build-essential python3 python3-pip python3-venv ffmpeg
03
Docker & n8n
6
Install Docker Engine
✓Docker avoids the OOM errors you get from global npm installs on t3.micro.
bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker ubuntu
newgrp docker
7
Run n8n container
bash
mkdir -p ~/.n8n

docker run -d \
  --name n8n \
  --restart unless-stopped \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=<STRONG_PASSWORD> \
  -e WEBHOOK_URL=https://n8n.yourdomain.com \
  n8nio/n8n:2.24.0
⚠Replace <STRONG_PASSWORD> with a real password. n8n listens internally on port 5678.
04
Nginx + Let's Encrypt SSL
8
Install Nginx & Certbot
bash
sudo apt install -y nginx certbot python3-certbot-nginx
sudo systemctl enable nginx
9
Create subdomain config — /etc/nginx/sites-available/n8n
nginx
server {
    listen 80;
    server_name n8n.yourdomain.com;

    location / {
        proxy_pass http://localhost:5678;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
    }
}
bash
sudo ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
10
Issue SSL certificate
⚠Point your domain A record to the Elastic IP before running certbot.
bash
sudo certbot --nginx -d n8n.yourdomain.com
# Certbot auto-edits nginx config & sets up auto-renewal
✓Repeat steps 9–10 for every subdomain: code. voice.
05
Hermes Agent
ℹHermes Agent is the NousResearch/hermes-agent framework — an AI agent orchestration gateway, not an LLM model. It exposes an OpenAI-compatible API and supports multiple LLM providers, STT via Groq, and TTS via Edge TTS.
11
Install Hermes CLI & initialise
bash
pip install hermes-agent --break-system-packages

# Initialise — creates ~/.hermes/
hermes init
12
Add an LLM provider — GitHub Copilot (OpenAI-compatible)

If you have a Copilot subscription, use it as a zero-cost LLM backend.

bash
hermes provider add gh_cli \
  --base-url https://api.githubcopilot.com \
  --token <YOUR_GHO_TOKEN> \
  --header 'Copilot-Integration-Id: vscode-chat' \
  --model claude-sonnet-4-5

hermes provider set-default gh_cli
⚠Token starts with gho_. Retrieve it with: gh auth token
13
Configure Groq for Speech-to-Text
~/.hermes/.env
GROQ_API_KEY=<YOUR_GROQ_API_KEY>
bash
hermes config set stt.provider groq
hermes config set stt.model whisper-large-v3
14
Run as a systemd user service

Create ~/.config/systemd/user/hermes.service:

ini
[Unit]
Description=Hermes Agent Gateway
After=network.target

[Service]
Type=simple
WorkingDirectory=%h/.hermes/hermes-agent
ExecStart=%h/.hermes/hermes-agent/venv/bin/python -m gateway.run
Restart=always
RestartSec=5
EnvironmentFile=%h/.hermes/.env

[Install]
WantedBy=default.target
bash
# Persist service across logouts
sudo loginctl enable-linger ubuntu

systemctl --user daemon-reload
systemctl --user enable hermes
systemctl --user start hermes
systemctl --user status hermes
✓Gateway on localhost:8642. Use hermes gateway restart after config changes.
⚠Auth broken? Run hermes debug share and check the paste URL — far more reliable than hermes auth status.
15
Expose Hermes via Nginx — /etc/nginx/sites-available/voice
nginx
server {
    listen 80;
    server_name voice.yourdomain.com;

    root /var/www/voice;
    index index.html;

    location /api/ {
        proxy_pass http://localhost:8642/;
        proxy_set_header Host $host;
        proxy_set_header Authorization 'Bearer <HERMES_API_KEY>';
    }
}
bash
sudo ln -s /etc/nginx/sites-available/voice /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d voice.yourdomain.com
06
Claude Code
16
Install Node.js via nvm
⚠Do not use npm install -g directly — it OOMs on t3.micro. nvm keeps Node local.
bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install --lts && nvm use --lts
17
Install Claude Code CLI & authenticate
bash
npm install -g @anthropic-ai/claude-code

# Start interactive auth — uses your Pro subscription
claude
07
Voice PWA
18
Create web root & deploy
bash
sudo mkdir -p /var/www/voice
sudo chown ubuntu:ubuntu /var/www/voice

# Copy your built PWA
cp -r dist/* /var/www/voice/
✓STT fix: call https://api.groq.com/openai/v1/audio/transcriptions directly from browser JS — Hermes lacks this endpoint.
